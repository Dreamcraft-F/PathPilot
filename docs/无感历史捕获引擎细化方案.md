# PathPilot 无感历史捕获引擎细化方案

## 1. 概述

无感历史捕获引擎是PathPilot的核心组件，负责在后台默默记录用户在Windows资源管理器中访问的路径。本方案细化了引擎的实现逻辑，包括记录触发条件、路径过滤规则、去重算法和智能识别策略。

## 2. 记录触发逻辑

### 2.1 事件监听机制
- **监听方式**：使用Windows UI Automation API监听资源管理器窗口焦点变化
- **触发条件**：当资源管理器窗口获得焦点（被激活）时触发路径读取
- **路径读取**：通过UI Automation获取资源管理器地址栏的当前路径
- **读取频率**：每次窗口焦点变化时读取，但应用去重逻辑避免重复记录

### 2.2 窗口识别
- **窗口类名**：识别资源管理器窗口类名（CabinetWClass）
- **排除干扰**：排除文件对话框、打开/保存对话框等非资源管理器窗口
- **多实例支持**：支持多个资源管理器窗口同时运行

## 3. 路径过滤规则

### 3.1 默认不记录的路径类型

#### 3.1.1 系统路径
- **Windows系统目录**：`C:\Windows\*`、`C:\Windows\System32\*`
- **程序安装目录**：`C:\Program Files\*`、`C:\Program Files (x86)\*`
- **系统隐藏目录**：`C:\$Recycle.Bin`、`C:\System Volume Information`

#### 3.1.2 临时路径
- **临时文件夹**：`C:\Temp\*`、`C:\Tmp\*`、`%TEMP%\*`
- **浏览器缓存**：`C:\Users\{用户}\AppData\Local\Google\Chrome\User Data\*`
- **下载临时文件**：`C:\Users\{用户}\Downloads\*.tmp`

#### 3.1.3 开发相关路径
- **版本控制目录**：`.git\*`、`.svn\*`、`.hg\*`
- **构建输出目录**：`build\*`、`dist\*`、`out\*`、`target\*`
- **依赖目录**：`node_modules\*`、`vendor\*`、`packages\*`

#### 3.1.4 应用数据路径
- **应用配置目录**：`C:\Users\{用户}\AppData\*`
- **应用缓存目录**：`C:\Users\{用户}\AppData\Local\*`

### 3.2 可配置的过滤规则
- **用户自定义排除**：支持用户通过JSON配置文件添加自定义排除路径
- **通配符支持**：支持`*`和`?`通配符匹配
- **正则表达式**：支持正则表达式匹配复杂路径模式

### 3.3 路径规范化
- **大小写统一**：Windows路径不区分大小写，统一转换为小写比较
- **路径分隔符**：统一使用`\`作为路径分隔符
- **去除尾部斜杠**：去除路径末尾的`\`符号
- **解析环境变量**：将`%USERPROFILE%`等环境变量解析为实际路径

## 4. 去重和智能过滤算法

### 4.1 基于时间的去重
- **短时间重复**：同一路径在5秒内多次访问，只记录第一次
- **时间窗口**：使用滑动时间窗口记录访问历史
- **实现方式**：维护一个路径→最后访问时间的映射表

### 4.2 基于层级的智能过滤

#### 4.2.1 路径层级分析
```
C:\Users\Documents\Project\src\components\Button.tsx
├── 层级0: C:\
├── 层级1: C:\Users
├── 层级2: C:\Users\Documents
├── 层级3: C:\Users\Documents\Project
├── 层级4: C:\Users\Documents\Project\src
├── 层级5: C:\Users\Documents\Project\src\components
└── 层级6: C:\Users\Documents\Project\src\components\Button.tsx
```

#### 4.2.2 向上导航过滤
- **场景**：用户从深层路径返回到父路径
- **示例**：`C:\A\B` → `C:\A` → `C:\A\B`
- **处理**：记录`C:\A\B`和`C:\A`，但标记`C:\A`为"导航路径"

#### 4.2.3 向下导航过滤
- **场景**：用户从父路径进入子路径
- **示例**：`C:\A` → `C:\A\B` → `C:\A\B\C`
- **处理**：记录`C:\A`和`C:\A\B\C`，中间路径`C:\A\B`标记为"过渡路径"

### 4.3 基于停留时间的过滤
- **短暂停留**：在某路径停留时间<2秒，可能是误操作或快速浏览
- **实现方式**：记录路径获得焦点和失去焦点的时间戳
- **阈值配置**：停留时间阈值可通过配置文件调整

### 4.4 基于访问模式的智能识别

#### 4.4.1 识别用户意图
- **探索模式**：用户快速浏览多个文件夹（间隔<5秒），可能在寻找文件
- **工作模式**：用户在特定文件夹停留较长时间（>30秒），可能在工作
- **返回模式**：用户频繁返回到某个路径，可能是重要位置

#### 4.4.2 路径重要性评分
```
分数计算公式：
得分 = 基础分 + 时间权重 + 频率权重 + 深度权重

基础分：默认100分
时间权重：停留时间越长，权重越高（最高+50分）
频率权重：访问次数越多，权重越高（最高+30分）
深度权重：路径深度适中（3-5层）的路径权重更高（±20分）
```

## 5. 数据结构设计

### 5.1 历史记录数据结构
```typescript
interface PathRecord {
  id: string;                    // 唯一标识符
  path: string;                  // 完整路径
  firstVisit: number;            // 首次访问时间戳
  lastVisit: number;             // 最后访问时间戳
  visitCount: number;            // 访问次数
  totalTimeSpent: number;        // 总停留时间（毫秒）
  importanceScore: number;       // 重要性评分
  isFavorite: boolean;           // 是否收藏
  tags: string[];                // 用户标签
  source: 'auto' | 'manual';    // 记录来源
  status: 'active' | 'archived' | 'deleted'; // 状态
}
```

### 5.2 访问事件数据结构
```typescript
interface VisitEvent {
  path: string;                  // 访问路径
  timestamp: number;             // 访问时间戳
  duration: number;              // 停留时间（毫秒）
  windowId: string;              // 窗口标识
  previousPath: string | null;   // 上一个路径
  navigationType: 'direct' | 'up' | 'down' | 'sibling'; // 导航类型
}
```

### 5.3 存储方案
- **主存储**：SQLite数据库，存储历史记录和访问事件
- **缓存**：内存缓存最近访问的100条记录，提高查询速度
- **备份**：每日自动备份数据库文件

## 6. 实现架构

### 6.1 模块划分
```
无感历史捕获引擎
├── 事件监听模块
│   ├── 窗口焦点监听器
│   ├── 路径读取器
│   └── 事件过滤器
├── 数据处理模块
│   ├── 路径规范化器
│   ├── 去重处理器
│   ├── 智能过滤器
│   └── 重要性计算器
├── 存储管理模块
│   ├── 数据库管理器
│   ├── 缓存管理器
│   └── 备份管理器
└── 配置管理模块
    ├── 规则配置加载器
    └── 用户设置管理器
```

### 6.2 工作流程
1. **事件捕获**：窗口焦点变化 → 读取路径 → 触发处理流程
2. **预处理**：路径规范化 → 应用过滤规则 → 检查是否记录
3. **去重处理**：检查时间去重 → 检查层级去重 → 检查模式去重
4. **数据存储**：计算重要性评分 → 更新数据库 → 更新缓存
5. **后处理**：清理过期数据 → 优化数据库 → 生成统计信息

### 6.3 性能优化
- **异步处理**：路径处理在后台线程执行，不阻塞UI
- **批量写入**：累积多个事件后批量写入数据库
- **延迟计算**：重要性评分在查询时计算，减少写入开销
- **内存管理**：限制缓存大小，定期清理过期数据

## 9. 注意事项

### 9.1 隐私保护
- **敏感路径**：自动识别并排除包含"password"、"secret"等关键词的路径
- **用户控制**：提供隐私模式开关，一键暂停记录
- **数据加密**：本地数据库使用简单加密保护

### 9.2 错误处理
- **权限问题**：无权限访问的路径静默跳过
- **路径不存在**：访问已删除的路径时标记为无效
- **网络路径**：网络路径超时时标记为离线

### 9.3 性能监控
- **资源占用**：监控CPU和内存使用，确保不影响系统性能
- **记录效率**：统计每秒处理路径数量，优化瓶颈
- **存储空间**：定期清理过期数据，控制数据库大小
